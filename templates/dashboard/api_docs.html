1. Getting an API key

Developers request a key; an admin approves it and the key is delivered over email and/or Telegram. Submit a request without authentication:

curl -X POST {{ request.scheme }}://{{ request.get_host }}/api/v1/request-key/ \ -H "Content-Type: application/json" \ -d '{ "name": "Acme Inc", "organization": "Acme", "email": "dev@acme.com", "telegram_username": "acme_dev", "reason": "Building a CRM integration", "requested_scopes": ["accounts:read","messages:read","messages:send"], "delivery_method": "both" }'

Once an admin approves it in Key requests, the key arrives via the chosen channel. It is shown only once.

2. Authentication

Send your key in the Authorization header on every request:

Authorization: Api-Key ubp_xxxxxxxx_your_secret # or X-API-Key: ubp_xxxxxxxx_your_secret

Each key carries scopes; endpoints require the matching scope.

4. Examples

# Send a message curl -X POST {{ request.scheme }}://{{ request.get_host }}/api/v1/accounts/1/send-message/ \ -H "Authorization: Api-Key $UBP_KEY" \ -H "Content-Type: application/json" \ -d '{"target": "@durov", "text": "Hello from the API"}' # Schedule a broadcast to all your accounts curl -X POST {{ request.scheme }}://{{ request.get_host }}/api/v1/schedules/ \ -H "Authorization: Api-Key $UBP_KEY" \ -H "Content-Type: application/json" \ -d '{"title":"Promo","text":"Big sale!","from_mode":"all", "targets":["@chan1","@chan2"],"scheduled_for":"2026-07-01T09:00:00Z"}' # Read your accounts curl {{ request.scheme }}://{{ request.get_host }}/api/v1/accounts/ \ -H "Authorization: Api-Key $UBP_KEY"

5. Privacy model

Accounts you connect can be marked private. Their message content is encrypted with your developer content key. The platform admin cannot read it from the panel without your API key — but every action (logins, edits, messages sent, channels created) is always recorded in the audit log. Through the API, with your key, you have full read/write access to your own accounts.
